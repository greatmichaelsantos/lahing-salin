Development Process

How We Built
SALIN-LAHI

A step-by-step story of how our heritage kiosk app was created — from idea to finished product.

Our Development Journey

Here is every step we took to build the SALIN-LAHI Heritage Interactive App.

1
📚
Research Phase
Gathered References and Conducted Research
We started by learning about the heritage and culture of Olongapo City. We gathered information from books, websites, and local resources to make sure our content was accurate and meaningful.
  • Researched the history of each heritage station
  • Collected photos and important dates for the timeline
  • Studied the indigenous culture of the Ayta Ambala people
  • Gathered information about local music, livelihoods, and traditions
2
🤖
Building Phase
Created the Program in HTML Using an AI Tool
We used an AI coding assistant (Claude Code by Anthropic) to help us write the code for our app. We described what we wanted and the AI helped turn our ideas into real working code using HTML, CSS, and JavaScript.
  • Designed the layout and color theme of the kiosk screen
  • Built the home dashboard with clickable heritage station tiles
  • Created the grid layout that fills the entire tablet screen
  • Used AI to speed up the coding process and fix errors
3
🎨
Design Phase
Designed the User Interface
We designed how the app looks and feels so it would be easy and fun for visitors to use at a kiosk. We chose colors, fonts, and layouts that matched the cultural theme of SALIN-LAHI.
  • Chose a blue and purple color scheme to represent culture and heritage
  • Added Google Fonts for a clean and modern look
  • Designed cards (tiles) for each heritage station on the home screen
  • Made sure everything fit nicely on a landscape tablet screen
4
🗄️
Data Phase
Added Content Using a Database
We used Firebase (a cloud database by Google) to store all the heritage content — the station descriptions, timeline events, quiz questions, and leaderboard scores. This means the content can be updated without changing the code.
  • Connected the app to Firebase Firestore (cloud database)
  • Stored heritage station info, timeline data, and quiz questions
  • Built an Admin Dashboard so content can be edited easily
  • Scores and leaderboard data saved in real time
5
🧩
Features Phase
Built the Interactive Features
We added special features to make the app interactive and engaging for users visiting the heritage kiosk.
  • Heritage station pages with photos, audio narration, and timelines
  • An interactive quiz where visitors can test their knowledge
  • A leaderboard to show top quiz scorers
  • Audio listening feature with a play button and progress bar
  • City Overview section with a map and key facts
  • Indigenous Culture modal with cultural stories and traditions
6
🧪
Testing Phase
Tested on a Real Tablet
After building the app, we tested it on an actual Huawei tablet (the same one used in the kiosk) to make sure everything worked properly on the real device — not just on a computer screen.
  • Opened the deployed app on a Huawei tablet using Vivaldi browser
  • Checked that all buttons, tiles, and panels displayed correctly
  • Found and reported bugs (things that looked wrong on tablet)
  • Took photos of issues and described them to the AI for fixing
7
🔧
Debugging Phase
Fixed Bugs and Improved the App
Every time we found a bug during testing, we worked with the AI tool to fix it. We went through many rounds of testing and fixing to make the app look and work perfectly.
  • Fixed buttons that were hidden or cut off on the tablet screen
  • Fixed the audio controls so they always stay visible
  • Fixed layout issues so tiles and text displayed side by side correctly
  • Adjusted sizes and spacing for the exact tablet resolution
8
🚀
Launch Phase
Deployed and Launched the Kiosk App
Once we were happy with how everything looked and worked, we deployed (published) the app to Firebase Hosting so it could be accessed on the kiosk tablet anytime — even without a laptop connected.
  • Published the app using Firebase Hosting
  • The tablet now loads the app directly from the cloud
  • The kiosk runs fully standalone — no laptop or extra computer needed
  • Visitors can now explore Olongapo's heritage at the kiosk!
